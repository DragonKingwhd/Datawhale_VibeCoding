# CLAUDE.md

本文件为 Claude Code (claude.ai/code) 在此仓库中工作时提供指导。

## 项目概述

**Datawhale_VibeCoding** 是一个学习记录仓库，用于记录 Datawhale VibeCoding 3 群的课程学习过程。

## 项目结构

项目采用任务驱动的组织方式，每个课程任务对应一个独立的文件夹：

```
Datawhale_VibeCoding/
├── task_01/
│   ├── post.md          # 任务说明和学习笔记
│   └── src/             # 任务对应的源代码
├── task_02/
│   ├── post.md
│   └── src/
├── task_03/
│   ├── post.md
│   └── src/
└── ...
```

### 文件夹命名规范

- 使用 `task_XX` 格式命名，其中 XX 为两位数字（如 `task_01`, `task_02`）
- 每个任务文件夹包含：
  - **post.md**: 任务说明、学习笔记、总结和心得
  - **src/**: 该任务对应的所有源代码文件

## 开发工作流

1. 为每个新任务创建对应的 `task_XX` 文件夹
2. 在 `post.md` 中记录任务说明和学习过程
3. 在 `src/` 目录中放置实现代码
4. 提交时使用清晰的提交信息，如 "Add task_01: 任务名称"
5. 每个项目只能够有一个最终的技术总结的md文件，不要有其他多余的md文件。

## 更新说明

- 随着项目的进展，根据实际情况更新此文件
- 如果任务涉及特定的编程语言或框架，在对应的 `post.md` 中记录相关的构建、运行和测试命令
